`PushUniversalAccountButton` is a versatile, state-aware button component for wallet connections in the Push Chain ecosystem. It handles the complete user journey from connection initiation through authentication to displaying the connected account state, with extensive customization options for each state.

## Usage

Place the button in your UI where users should be able to connect. The button must be used within a `PushUniversalWalletProvider`:

## Button States

The button transitions between three main states:

1. **Disconnected**: Login button prompting users to connect
2. **Loading**: Connection/authentication in progress
3. **Connected**: Displays the connected account with dropdown menu

Each state can be fully customized through props.

## Props

### Instance Targeting

<PushAPIReference showRequiredNotice={false}>

| Arguments | Type     | Default               | Description                                                                                                                                         |
| --------- | -------- | --------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| `uid`     | `string` | Uses default provider | Should match `config.uid` of specific `PushUniversalWalletProvider` instance.<br />&nbsp;<br />See [multiple wallet example](/docs/chain/ui-kit/examples/multiple-wallet-example/) for usage. |

</PushAPIReference>

### Disconnected State Customization

<PushAPIReference showRequiredNotice={false}>

| Property            | Type     | Default           | Description                 |
| ------------------- | -------- | ----------------- | --------------------------- |
| `connectButtonText` | `string` | `Connect Account` | Text for the connect button |

</PushAPIReference>

### Loading State Customization

<PushAPIReference showRequiredNotice={false}>

| Property           | Type              | Default             | Description              |
| ------------------ | ----------------- | ------------------- | ------------------------ |
| `loadingComponent` | `React.ReactNode` | Uses default loader | Custom loading indicator |

</PushAPIReference>

### App Preview Overrides

<PushAPIReference showRequiredNotice={false}>

| Property                       | Type     | Default                | Description                                |
| ------------------------------ | -------- | ---------------------- | ------------------------------------------ |
| `modalAppOverride`             | `Object` | From `app`             | Override app preview in the wrapper iframe |
| `modalAppOverride.logoUrl`     | `string` | From `app.logoUrl`     | Override app logo                          |
| `modalAppOverride.title`       | `string` | From `app.title`       | Override app title                         |
| `modalAppOverride.description` | `string` | From `app.description` | Override app description                   |
| `loginAppOverride`             | `Object` | From `app`             | Override app preview inside hosted iframe  |
| `loginAppOverride.logoUrl`     | `string` | From `app.logoUrl`     | Override app logo in iframe                |
| `loginAppOverride.title`       | `string` | From `app.title`       | Override app title in iframe               |
| `loginAppOverride.description` | `string` | From `app.description` | Override app description in iframe         |

</PushAPIReference>

### Button Theme Settings Override

<PushAPIReference showRequiredNotice={false}>

| Property         | Type             | Default | Description            |
| ---------------- | ---------------- | ------- | ---------------------- |
| `themeOverrides` | `ThemeOverrides` | `{}`    | Button theme overrides |

</PushAPIReference>
