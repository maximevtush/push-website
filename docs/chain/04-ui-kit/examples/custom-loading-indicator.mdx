# Custom Loading Indicator

This example demonstrates how to create and integrate a custom loading indicator with the `PushUniversalAccountButton` component.

## Basic Implementation

<Tabs className="liveplaytab" groupId="custom-loading-indicator">
  <TabItem value='basic' label='Basic'>

```jsx live
import {
  PushUniversalWalletProvider,
  PushUniversalAccountButton,
  PushUI,
} from '@pushchain/ui-kit';

function App() {
  // Simple custom loading component
  const SimpleLoader = () => (
    <div style={{ display: 'flex', alignItems: 'center', gap: '8px' }}>
      <div
        style={{
          width: '16px',
          height: '16px',
          borderRadius: '50%',
          border: '3px solid rgba(0, 0, 0, 0.1)',
          borderTopColor: '#6B46C1',
          animation: 'spin 1s linear infinite',
        }}
      />
      <style jsx>{`
        @keyframes spin {
          to { transform: rotate(360deg); }
        }
      `}</style>
      <span>Loading wallet...</span>
    </div>
  );

  return (
    <PushUniversalWalletProvider
      config={{ network: PushUI.CONSTANTS.PUSH_NETWORK.TESTNET }}
    >
      <PushUniversalAccountButton 
        loadingComponent={<SimpleLoader />}
        connectButtonText="Connect with Custom Loader"
      />
    </PushUniversalWalletProvider>
  );
}
```

## Implementation Details

The custom loading indicator is implemented as a React component and passed to the `loadingComponent` prop of the `PushUniversalAccountButton`. This allows you to:

1. Replace the default loading spinner with your own design
2. Match your application's branding and style
3. Provide more context to users during the loading state

### Key Points

- The loading component is rendered when the wallet connection is in progress
- You can use CSS animations for smooth visual feedback
- The component can include both visual elements and text
- You can use Push brand colors for a consistent look and feel

## Usage with Multiple Wallet Instances

When using multiple wallet instances, make sure to specify the `uid` prop to target the specific wallet instance:

```jsx
<PushUniversalWalletProvider
  config={{ 
    network: PushUI.CONSTANTS.PUSH_NETWORK.TESTNET,
    uid: 'wallet-1'
  }}
>
  <PushUniversalAccountButton 
    uid="wallet-1"
    loadingComponent={<PushBrandedLoader />}
  />
</PushUniversalWalletProvider>
```

## Best Practices

1. Keep the loading indicator simple and lightweight
2. Provide visual feedback that indicates progress
3. Match your application's design language
4. Consider accessibility - ensure the loading state is perceivable to all users
5. Add appropriate text to inform users what is happening
