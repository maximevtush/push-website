## Overview

Initializing the SDK client gives you:

- A **chain-agnostic** `PushChainClient` for submitting on-chain calls  
- Automatic **RPC endpoint** resolution (plus optional overrides)  
- Built-in **UEA** (Universal Executor Account) and **origin** account getters  
- Cross-chain **gas abstraction**, signature orchestration, and debug traces  

Just pass your universal signer and network, and youâ€™re ready to write and transact on Push Chain from any wallet.

## Initialize Push Chain Client

**_`PushChain.initialize(signer, {options}): Promise<PushChainClient>`_**

```typescript
// Import @pushchain/core and ethers
// Ensure you have created Universal Signer. 
// If not done, then check out Create Universal Signer.

// Initialize Push Chain Client
const pushChainClient = await PushChain.initialize(universalSigner, {
  network: PushChain.CONSTANTS.PUSH_NETWORK.TESTNET,
});
```

| Arguments                 | Type                             | Default                                          | Description                                                                                                                                                                                          |
| --------------------- | -------------------------------- | ------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| _`signer`_   | `UniversalSigner`                | -                                                | Chain-agnostic signer responsible for signing transactions and messages.                                                                                                                         |
| _`options.network`_     | `PushChain.CONSTANTS.PUSH_NETWORK`                        | - | Push Chain network to conenct to. For example: `PushChain.CONSTANTS.PUSH_NETWORK.TESTNET` <Details summary="PushChain.CONSTANTS.PUSH_NETWORK" className="alert alert--minimal-api-table">`PushChain.CONSTANTS.PUSH_NETWORK.TESTNET` `PushChain.CONSTANTS.PUSH_NETWORK.TESTNET_DONUT` `PushChain.CONSTANTS.PUSH_NETWORK.LOCALNET`</Details>                                                                                                                                |
| `options.rpcUrls`     | `Partial<Record<CHAIN, string>>` | `{}`                                             | Custom RPC URLs mapped by chain IDs. If not provided, the default RPC URLs for the network will be used. Example: `rpcUrls: {[CHAIN.ETHEREUM_SEPOLIA]: 'https://sepolia.infura.io/v3/your-api-key', [CHAIN.SOLANA_DEVNET]: 'https://api.devnet.solana.com'}` |
| `options.blockExplorers` | `Partial<Record<CHAIN, string[]>>` | `{[CHAIN.PUSH_TESTNET_DONUT]: ['https://donut.push.network']}` | Custom block explorer URLs mapped by chain IDs. If not provided, the default block explorer URLs for the network will be used. |
| `options.printTraces` | `boolean`                        | `false`                                          | When true, console logs the internal trace logs for debugging requests to nodes                                                                                                                |

## Get Push Chain Client UEA and Universal Origin Account

The Push Chain Client provides access to two important accounts: the UEA (Universal Executor Account) and the Universal Origin Account. The UEA is the smart contract account on Push Chain that executes transactions when user is using fee abstraction, while the Universal Origin Account represents the Universal Account of the user's wallet on the blockchain network where the user's wallet originates from (e.g., Ethereum Sepolia, Solana Devnet, etc.).

```typescript
// UEA Address
const ueaAddress = pushChainClient.universal.account

// Origin Universal Account
const originAccount = pushChainClient.universal.origin
```

## Creating Signatures with Push Chain SDK

The Push Chain SDK provides three essential signature functions that work across different blockchain networks (EVM and SVM). These functions handle the complexity of signing data and transactions in a chain-agnostic manner.

### Available Signature Functions

| Function | Description | Return Type |
|----------|-------------|-------------|
| `signMessage` | Signs arbitrary message data | Hex string |
| `signTypedData` | Signs structured data (EIP-712) | Hex string |

### 1. Sign Message

The `signMessage` function signs arbitrary message data and automatically handles the encoding format.

```typescript
// Create message data
const message = new TextEncoder().encode('Hello, Push Chain!')

// Sign the message
const signature = await pushChainClient.universal.signMessage(message)
```

### 2. Sign Typed Data

The `signTypedData` function signs structured data following the EIP-712 standard. This function is primarily used for EVM chains and throws an error if the underlying signer doesn't support typed data signing.

```typescript
// EIP-712 typed data structure
const domain = {
  name: 'Push Chain',
  version: '1',
  chainId: 42101, // Push testnet
  verifyingContract:
    '0x1234567890123456789012345678901234567890' as `0x${string}`,
};

const types = {
  Person: [
    { name: 'name', type: 'string' },
    { name: 'wallet', type: 'address' },
  ],
};

const message = {
  name: 'Alice',
  wallet: '0x9821655B609186a9296261638FA74e1DFBA4AC88' as `0x${string}`,
};

// Sign the typed data
const signature = await pushChainClient.universal.signTypedData({
  domain,
  types,
  primaryType: 'Person',
  message,
})
```
